Static RUN_EVERY macro with second time interval. Marks the start of the code block.

Note
Use this macro when you don't want to deal with chrono object instantiation.
This macro does not execute user code during the first encounter of the code block. It only initializes the chrono object and puts it in the run state. User code will be executed in the subsequent intervals.
It is essential to mark the end of the code block using the RUN_END_ macro.
Parameters
name_Name of the code block.
intervalUs_The time interval for running the code block in seconds.

#define RUN_EVERY_S_OBJ_ ( name_,
chrono_,
intervalS_ )   RUN_EVERY_BASE_(name_, intervalS_, chrono_, S, false)

Objective RUN_EVERY macro with second time interval. Marks the start of the code block.

Note
Use this macro when you have already created a chrono object. Since RUN_EVERY_S_() creates its chrono object statically, users can't use it when an object-oriented approach is used to write the code. For example, in a method of a class, users can't use RUN_EVERY_S_() because all instances of that class will share a static chrono object. Instead of using RUN_EVERY_S_(), utilize RUN_EVERY_S__OBJ_() to pass the name of the chrono object.
This macro does not execute user code during the first encounter of the code block. It only initializes the chrono object and puts it in the run state. User code will be executed in the subsequent intervals.
It is essential to mark the end of the code block using the RUN_END_ macro.
Parameters
name_Name of the code block.
chrono_Pointer to the chrono object.
intervalUs_The time interval for running the code block in seconds.
